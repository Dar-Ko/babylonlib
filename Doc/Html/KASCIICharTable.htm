American Standard Code for Information Interchange (ASCII1) Table

Dec Hex Char HTML Description
0 0 ^@ NUL ␀ Null Used for time delay or to fill space when there is no data.
1 1 ^A SOH ␁ Start of heading Indicates the start of a heading which may contain address or routing information.
2 2 ^B STX ␂ Start of text Indicates the end of the heading and the start of the text.
3 3 ^C ETX ␃ End of text Used to terminate the text which was started with STX.
4 4 ^D EOT ␄ End of transmission Indicates the end of a transmission of one or more text blocks.
5 5 ^E ENQ ␅ Enquiry A request for a response from a remote station.
6 6 ^F ACK ␆ Acknowledge An affirmation response to a sender.
7 7 ^G BEL ␇ Bell/ Alert/ Beep Used when there is need to call human attention. It may control alarm or attention devices.
8 8 ^H BS ␈ Backspace Indicates movement of the printing mechanism or display cursor backwards in one position.
9 9 ^I HT/ TAB ␉ Horizontal tab Indicates movement of the printing mechanism or display cursor forward to the next 'tab' or stopping position.
10 A ^J LF/ NL ␊ Line feed/ New line Indicates movement of the printing mechanism or display cursor to the start of the next line (one line down).
11 B ^K VT ␋ Vertical tab/ Home Indicates movement of the printing mechanism or display cursor to the next of a series of printing lines.
12 C ^L FF/ NP ␌ Form feed/ New page Indicates movement of the printing mechanism or display cursor to the starting position of the next page, form, or screen.
13 D ^M CR ␍ Carriage return Indicates movement of the printing mechanism or display cursor to the starting position (left) of the current line.
14 E ^N SO ␎ Shift out Indicates that the code combinations which follow shall be interpreted as differently than in standard set until an SI character is reached.
15 F ^O SI ␏ Shift in Indicates that the code combinations which follow shall be interpreted according to the standard character set.
16 10 ^P DLE ␐ Data link escape  
17 11 ^Q DC1/ XON ␑ Device control 1 Characters for the control of ancillary devices or special terminal features like XON in communication handshaking 
18 12 ^R DC2 ␒ Device control 2 Characters for the control of ancillary devices or special terminal features.
19 13 ^S DC3/ XOFF ␓ Device control 3 Characters for the control of ancillary devices or special terminal features like XOFF in communication handshaking.
20 14 ^T DC4 ␔ Device control 4 Characters for the control of ancillary devices or special terminal features.
21 15 ^U NAK ␕ Negative acknowledge A negative response to a sender sent by a receiving device.
22 16 ^V SYN ␖ Synchronous/ Idle Used  to achieve synchronization  or to signal that no data pending in synchronous communication.
23 17 ^W ETB ␗ End of transmission block Indicates the end of a block of data for communication purposes. It is used for blocking data where the block structure is not necessarily related to the processing format.
24 18 ^X CAN ␘ Cancel Indicates that the previously sent data should be disregarded.
25 19 ^Y EM ␙ End of medium Indicates the physical end of a card, tape or other medium, or the end of the required or used portion of the medium.
26 1A ^Z SUB ␚ Substitute Substituted for a character that is found to be erroneous or invalid.
27 1B ^[ ESC ␛ Escape Indicates the start of character sequence with special meaning.
28 1C ^\ FS ␜ File separator/ Cursor right  
29 1D ^] GS ␝ Group separator/ Cursor left  
30 1E ^^ RS ␝ Record separator/ Cursor up  
31 1F ^_ US ␟ Unit separator/ Cursor down

Legend: ASCII Control Character Codes

Physical Device Controls: Format Effectors

  • BS: Backspace
  • HT: Horizontal Tab
  • LF: Line Feed
  • VT: Vertical Tab
  • FF: Form Feed
  • CR: Carriage Return

Logical Communications Controls

  • SOH: Start of Header
  • STX: Start of Text
  • ETX: End of Text
  • EOT: End of Transmission
  • ENQ: Enquiry
  • ACK: Acknowledge
  • DLE: Data Link Escape
  • NAK: Negative Acknowledge
  • SYN: Synchronous Idle
  • ETB: End of Transmission Block

Physical Device Controls: Other

  • BEL: Bell
  • DC1: No assigned meaning in ASCII
  • DC2: No assigned meaning in ASCII
  • DC3: No assigned meaning in ASCII
  • DC4: No assigned meaning in ASCII

Information Separators

  • FS: File Separator
  • GS: Group Separator
  • RS: Record Separator
  • US: Unit Separator

Physical Communications Controls

  • NUL: Null
  • DEL: Delete
  • CAN: Cancel
  • EM: End of Medium
  • SUB: Substitute

Controls for Code Extensions

  • SI: Shift In
  • SO: Shift Out
  • ESC: Escape
