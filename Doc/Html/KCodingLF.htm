Coding Look and Feel


C/C++ Library
  • Coding standard 1 is very useful when more than two people are involved on the same project. It easier to analyze and maintain the code, if code appearance is somewhat uniform. 80% of the lifetime cost of a software goes to maintenance.
  • Source code indentation may contain tab characters, but the code formatter will convert it to spaces. Tabs are inconvenient for large group of developers because they don't necessarily look the same in every editing environment. Modern editors let you compose code just as easily with spaces as with tabs. All blocks of code (functions, loops, if's, etc.) will be indented two spaces from opening curly brace '{' to closing curly brace '}'.  Opening and closing braces will be placed on a separate line and indented to the same column.
  • Variable names should follow at least rudimentary Hungarian Notation2. If you're maintaining someone else's code, or even your own several years later, following the logic behind the code is error-prone. If you could readily recognize variable type, analyzing the code becomes little easier. Even if you are working with strong-typed language, where compiler will warn you about inappropriate type casts, Hungarian Notation2 is helpful when you are reading lengthy printouts.
  • Variable scope should be indicated with common prefixes like following: g_ for global variables, s_ for static or m_ for class members.
  • Global and module specific (static) variables should be avoided whenever possible. If data needs to be shared across functions (and handlers), space for it should be allocated on the heap and registered as thread specific data. This helps avoid common multithreading bugs.
  • Shared variables which are passed between threads may need to be declared as volatile 3 in order to prevent the compiler from using stale data cached in registers.  Unfortunately, the definition of volatile differs form compiler to compiler, but it is a good idea to declare a variable as volatile if it is changed before a synchronization point and referenced after a synchronization point during which another thread may have modified the value.
  • Line length should be 80 characters, since longer lines are not handled well by many terminals and tools.

1 Extreme Programming:
Coding Standard - The programmers all write code in the same way. This allows them to work in pairs and to share ownership of the code.
Coding standards typically address not only style, but also naming (Hungarian notation).

2 Hungarian Notation is a variable name prefixed with a common tag so you know what data type a variable name is carrying. It is invented to help programmers know what the data type was they were working with, to prevent  bad assignment operations in weak-typed languages such as C.

3volatile declarator is a type qualifier used to declare that an object can be modified in the program by something other than statements, such as the operating system, the hardware, or a concurrently executing thread.
