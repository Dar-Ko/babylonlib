Microsoft Database Connectivity

Database Connection Strings

ODBC2 driver connection string format:

"Keyword1=value1;Keyword2=value2; ... KeywordN=valueN"

Example:
"Driver=DriverName;Server=DbServer;Database=DbName; Uid=AuthorizationName;Pwd=AuthenticationString;"

The following table lists common connection string keywords that are recognized by an ODBC driver:

Table 1: Microsoft ODBC Driver Keywords
Keyword Description
DSN5 ODBC data source name is an alias name of the database pointing to required connection information
Driver name of the ODBC driver that you want to use, to be used only if DSN is not specified
UserID or UID authorization name (user identifier) specifies the user identity
Password or Pwd authentication string specifies user's password

OLE DB3 provider connection string format:

"Keyword1=value1;Keyword2=value2; ... KeywordN=valueN"

Example:
"Provider=ProviderName;Driver=DriverName;Server=DbServer;Database=DbName; Uid=AuthorizationName;Pwd=AuthenticationString;"
