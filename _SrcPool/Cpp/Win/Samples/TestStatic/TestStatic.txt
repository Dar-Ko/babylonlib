If linker generates the following error, when you build an MFC application with UNICODE:
  Microsoft Visual C++ 6.0 or less: 
  error LNK2001: unresolved external symbol "_WinMain@16"

change Project Settings | Link | Output category | Entry Point Symbol to wWinMainCRTStartup.

  Microsoft Visual C++ .NET
  error LNK2019: unresolved external symbol _WinMain@16 referenced in function _WinMainCRTStartup

change Project Properties | Linker | Advanced | Entry Point to wWinMainCRTStartup.

Ref: MSDN Article Q125750


warning LNK4075: ignoring '/EDITANDCONTINUE' due to '/INCREMENTAL:NO' specification
  Microsoft Visual C++ 6.0 or more
feature can be enabled/disabled by Tools | Options | Debug | "Debug commands invoke Edit and Continue" check box.
